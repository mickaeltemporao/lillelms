## 3. Sujets prioritaires à aborder

### A. Fondamentaux (transversaux)
1. **Concepts clés** : LLM, tokens, température, prompt engineering ; différence LLM vs modèles classiques (classification, extraction) vs fine-tuning
2. **Panorama des outils** : modèles gratuits (ChatGPT API, Mistral, Llama via Ollama) vs propriétaires vs hébergés localement ; quand utiliser quoi
3. **Chaînes de traitement** : pipeline typique (nettoyage → chunking → embedding → LLM → post-traitement)
4. **Enjeux éthiques & RGPD** : anonymisation préalable, confidentialité données sensibles, biais algorithmiques, traçabilité et citabilité des résultats
5. **Reproductibilité & critiques** : comment justifier méthodologiquement l'usage d'IA ; documenter prompts et versions modèles ; mesurer fiabilité

### B. Pratique : par type de données

#### **Corpus textuels structurés** (priorité : [1, 5, 9, 21, 26])
- Extraction d'entités nommées (fonctions, institutions, thèmes) via NER ou prompts
- Classification assistée (tonalité, catégorie, stance) : comparaison humain vs LLM
- Résumé et synthèse : quand c'est robuste, quand ça échoue
- Similarité sémantique et clustering texte (embeddings) pour regrouper documents thématiquement
- **Application prioritaire** : analyser 300 PLU [21] via extraction de règles + embeddings sémantiques pour dépasser formulations hétérogènes

#### **Données multimodales** (priorité : [3, 6, 11, 27])
- Vision-language models (GPT-4V, Claude) pour analyser images/vidéos + texte
- Cas d'usage : annotation d'étiquettes [6] ; réaction comments ↔ visuels posts [3, 11]
- Limites : coûts API, latence, traçabilité
- **Application prioritaire** : comparing Goodreads reviews × book covers + publisher summaries [27]

#### **Données audio** (priorité : [8, 10, 26])
- Transcription (Whisper) + traduction si multilingue [20]
- Chaînes de coréférence, entités, événements : pipeline après transcription
- **Application prioritaire** : réunions hybrides [10] ; entretiens doctorants [26]

#### **Web scraping & APIs** (priorité : [3, 5])
- Outils : Selenium, BeautifulSoup, APIs officielles (problèmes d'accès)
- Nettoyage post-scrape : HTML cleanup, déduplication
- Rythme & éthique : pas de surcharge serveur ; respect ToS
- **Application prioritaire** : X/TikTok API + commentaires [3] ; Tribunal decisions [5]

#### **Données quantitatives + qualitatives** (priorité : [11, 14, 18, 22, 25])
- Imputations assistées par LLM (contexte + logique)
- Segmentation/clustering hybride (features quantiques + représentations textuelles)
- Génération d'hypothèses exploratoires à partir de patterns
- **Application prioritaire** : 200k commentaires Facebook [11] → LLM classification vs human coding ; validation et mesures d'accord

### C. Usages avancés (pour intervenants experts)

#### **Fine-tuning & RAG** (pour [5, 14, 18, 19, 21])
- Quand fine-tuner vaut-il le coup ? (données, coûts, gains)
- Retrieval-Augmented Generation : injecter contexte spécifique (archives judiciaires, corpus propre) sans fine-tuning
- **Cas d'usage** : répondre à requêtes spécifiques sur base documentaire confidentielle [5, 26]

#### **Modèles spécialisés** (pour linguistes & SHS)
- CamemBERT, mBERT, XLM-R pour français + multilingue [20, 23]
- Transformers pour analyse sémantique structurée [21]
- Quand utiliser embeddings contextuels vs word2vec
- **Application** : détecter formulations équivalentes en droit [9, 21] ; analyse cross-lingue [20]

#### **Automatisation workflows** (pour [18, 19])
- Orchestration (Make.com, n8n) pour pipeline data → LLM → output
- Batch processing via APIs pour volumétries importantes (>10k documents)
- **Cas d'use** : anonymisation + codage + reporting automatisé

### D. Réflexions critiques & méthodologiques

1. **Validation des résultats** [11, 13, 15, 19, 25]
   - Sampling et test sur sous-corpus avant déploiement complet
   - Accord inter-codeurs (humain vs IA vs humain + IA)
   - Mesures d'incertitude et gestion erreurs LLM
   
2. **Biais & limitations** [14, 19, 25, 26, 28]
   - Biais inhérents aux modèles (construction, données d'entraînement)
   - Hallucinations LLM et stratégies de mitigation
   - Représentation inégale de certains contextes (langue, géographie, domaine)
   
3. **Traçabilité & citabilité** [5, 8, 18, 25, 28]
   - Documenter version modèle, prompts, température, seed
   - Reproductibilité = requête achèverait même résultat ?
   - Standards de citation pour LLM en sciences sociales
   
4. **Données sensibles** [10, 28] (entretiens, données identifiantes)
   - Anonymisation **avant** utilisation IA (ne pas dépendre de confidentialité du provider)
   - Clauses contractuelles (pas de training sur données)
   - Alternatives : modèles locaux, open-source
